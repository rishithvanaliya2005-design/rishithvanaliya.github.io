Goals

Architecture

  • Frontend: (HTML/JS/CSS) — UI pages and file upload
  • Backend: Flask app — endpoints to create case, trigger meshing, run solver, postprocess
  • CFD engine: OpenFOAM — template cases + parameter substitution
  • Job management: Celery / systemd / simple subprocess queue (describe what you used)
  • Storage: case directory layout, results retention

Usage

  1. Upload geometry (STL) or choose parametric shape
  2. Specify BCs (inlet velocity, outlet, walls)
  3. Choose simulator (simpleFoam, pisoFoam) and mesh parameters
  4. Run and wait — status visible on UI; download logs & result archives.
